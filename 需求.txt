产品需求文档 (PRD)：AesthetiCore —— 医疗美容诊所一体化 AI 智能操作系统
1. 执行摘要与战略愿景
1.1 产品愿景
中国医疗美容行业正处于从粗放式营销驱动向精细化运营与合规驱动转型的关键时期。当前市场虽保持着约 14.25% 的年复合增长率（CAGR），预计到 2032 年市场规模将达到 181.17 亿美元 ，但行业内部却面临着严峻的“盈利悖论”。尽管营收增长，但由于极高的获客成本（CAC）、昂贵的医疗设备维护费以及专业医护人员的高薪酬，许多诊所的净利润率已被压缩至 10% 以下，甚至面临亏损 。
在此背景下，"AesthetiCore" 系统不仅仅是一套软件工具，更被定义为医疗美容机构的“智能中枢”。其核心愿景是通过 “规模化的精准智能” (Precision at Scale) 解决当前诊所运营中的碎片化痛点。系统深度融合了计算机视觉（CV）、生成式人工智能（AIGC）与预测性分析技术，旨在实现从术前模拟、品牌营销、治疗方案设计到财务风控的全链路智能化闭环。我们的目标是将机构从繁琐的低效劳动中解放出来，让医生回归医疗本质，让运营回归数据理性，通过技术手段重建医美行业的信任机制与利润结构。
1.2 战略背景与商业必要性分析
根据深入的市场调研与数据分析，开发 AesthetiCore 系统的必要性主要体现在以下三个维度的迫切需求：
 * 信任构建与转化率提升（Trust Gap Bridge）：
   在医美消费决策中，最大的阻碍是“不确定性”。消费者对于术后效果的担忧直接导致了决策周期的延长和转化率的低下。现有的模拟工具往往过于依赖简单的 2D 液化滤镜，缺乏解剖学依据，不仅无法真实反映玻尿酸填充的体积感，甚至可能因过度承诺导致医疗纠纷 。AesthetiCore 提出的 FaceSim 模块，特别是针对玻尿酸和祛痘祛斑的模拟，必须基于严谨的 3D 形态学模型和皮肤纹理生成对抗网络（GAN），以提供“所见即所得”的医学级模拟，从而降低患者的决策焦虑。
 * 运营成本结构的重构（Cost Structure Optimization）：
   传统医美机构在内容生产和数据分析上投入了大量的人力成本。一家中型诊所通常需要配备专门的平面设计师、文案策划和财务分析师。然而，设计产出的海报往往缺乏品牌一致性，财务报表由于数据滞后而无法指导实时决策。AesthetiCore 通过 BrandGuard（品牌卫士） 和 ReportFlow（智能报表） 模块，利用 AI 自动化替代重复性脑力劳动，直接降低人力运营成本（OPEX），使机构能够将资源更多地投入到核心医疗服务中 。
 * 客单价与生命周期价值的最大化（Revenue Intelligence）：
   目前大部分诊所的定价策略仍停留在静态价目表阶段，缺乏对供需关系、医生职级差异及客户支付能力的动态感知。通过 ConsultAI（咨询大脑） 模块引入动态定价（Dynamic Pricing）和智能配置报价（CPQ）逻辑，系统能够根据实时库存、医生排班饱和度及客户画像，生成最优的个性化治疗方案，从而在保证客户满意度的前提下最大化单客价值（ARPU）。
1.3 本文档范围
本文档详细规定了 AesthetiCore 平台的功能性需求（Functional Requirements）与非功能性需求（Non-Functional Requirements），涵盖了从底层算法选型、数据流转逻辑到前端用户交互的全方位设计。本文档将作为研发团队、数据科学团队、UI/UX 设计团队及测试团队工作的核心依据。
系统核心功能模块概览：
 * AI 美学模拟器 (FaceSim Module)： 涵盖 2D 皮肤问题处理与 3D 面部容量模拟。
 * 品牌一致性设计引擎 (BrandGuard Module)： 基于 VI 体系的自动化海报生成。
 * 自动化汇报套件 (ReportFlow Module)： 智能数据聚合与 PPT 生成。
 * 智能治疗方案与定价系统 (ConsultAI Module)： 动态 CPQ 引擎与个性化方案推荐。
 * 财务智能仪表盘 (FinOps Module)： 成本结构分析与运营建议。
 * 生成式视频工坊 (MediaGen Module)： 专业短视频脚本与画面生成。
2. 系统架构与技术底座
2.1 用户画像与权限体系
为确保系统设计的针对性，我们定义了以下四类核心用户角色，并针对其工作流特征进行功能匹配：
 * 美学咨询师 (The Aesthetic Consultant)：
   * 核心痛点： 沟通效率低，缺乏直观工具辅助成交，手动计算报价易出错。
   * 系统赋能： 需提供毫秒级的术后模拟反馈，以及一键生成的报价单，支持移动端（iPad）操作以适应咨询场景。
 * 整形外科/皮肤科医生 (The Medical Practitioner)：
   * 核心痛点： 解释专业术语困难，缺乏高质量的科普素材，由于行政工作挤占临床时间。
   * 系统赋能： 需提供解剖学准确的 3D 模型辅助讲解，以及自动化的视频生成工具用于个人 IP 建设。
 * 诊所经营者/院长 (The Clinic Manager/Owner)：
   * 核心痛点： 无法实时掌握经营状况，数据孤岛导致决策滞后，利润率不可控。
   * 系统赋能： 需提供可视化的 BI 仪表盘，关注利润率、人效坪效及库存周转，并提供 AI 驱动的运营诊断建议。
 * 市场营销专员 (The Marketing Specialist)：
   * 核心痛点： 内容产出压力大，不仅要保证数量还要符合品牌调性，合规风险高。
   * 系统赋能： 提供严格遵循 VI 规范的海报生成器和文案辅助工具，内置广告法违禁词审查机制。
2.2 总体架构设计
AesthetiCore 将采用 云原生微服务架构 (Cloud-Native Microservices Architecture)，以确保系统的高可用性、可扩展性及数据的安全性。考虑到涉及高算力的 AI 图形处理任务，我们将采用计算分离的策略。
2.2.1 基础设施层 (Infrastructure Layer)
 * 混合云部署： 考虑到中国《数据安全法》对医疗数据本地化的要求 ，核心数据库及包含个人敏感信息（PII/PHI）的服务将部署在阿里云或腾讯云的中国境内节点。对于非敏感的通用模型训练任务，可利用全球分布式 GPU 算力资源。
 * GPU 算力池：
   * 训练集群： 部署 NVIDIA A100/H100 显卡集群，用于定期的模型微调（Fine-tuning），如针对特定品牌风格的 LoRA 模型更新或皮肤病理数据集的 GAN 模型迭代 。
   * 推理集群： 部署 NVIDIA T4 或 L4 显卡，用于实时的图像生成和 3D 渲染请求。T4 显卡在保持较低成本的同时，能够提供足够的 FP16 算力以满足 2D/3D 图像推理的低延迟需求 。
2.2.2 核心服务层 (Core Services Layer)
系统逻辑被拆分为独立部署的微服务，通过 API Gateway 进行统一流量调度：
 * FaceSim Engine（模拟引擎）： 封装了 StyleGAN3、SkinGAN 及 3DMM 重建算法，负责所有图像处理任务。
 * Design Graph Service（设计图谱服务）： 集成了 Stable Diffusion XL (SDXL) 与矢量图形处理库，负责海报与视觉物料的生成。
 * Analytics Core（分析核心）： 基于 Python Pandas/NumPy 的数据处理单元，负责财务数据清洗、KPI 计算及预测模型运行。
 * Content Brain（内容中枢）： 接入大语言模型（如 GPT-4o 或国内的通义千问/文心一言），负责文案生成、视频脚本编写及自然语言交互。
2.2.3 数据持久化层 (Data Persistence Layer)
 * 关系型数据库 (PostgreSQL)： 存储用户数据、订单信息、财务记录及系统配置。
 * 向量数据库 (Vector DB - Milvus/Pinecone)： 存储面部特征向量、设计素材的语义向量，支持以图搜图及相似案例推荐。
 * 对象存储 (OSS/S3)： 存储海量的原始高清图片、生成的过程图及视频文件，需配置生命周期管理以优化存储成本。
2.3 安全与合规性架构
鉴于医美行业的特殊性，系统必须内置严格的安全机制：
 * 数据脱敏 (De-identification)： 在将数据输入到任何公共 AI 模型进行处理前，必须通过本地部署的轻量级模型对人脸关键特征进行不可逆的特征扰动或遮盖，确保输出结果无法反向追踪到具体自然人，除非获得显式授权 。
 * 访问控制 (RBAC)： 基于角色的访问控制策略，严格限制只有主治医生及授权咨询师才能查看未脱敏的完整病历照片。
 * 水印系统 (Digital Watermarking)： 所有系统生成的模拟图和海报均自动嵌入不可见的数字水印，包含生成时间、操作员 ID 及机构 ID，以防止图像被滥用或作为虚假广告传播。
3. 模块一：AI 美学模拟器 (FaceSim Module)
3.1 需求深度拆解与算法选型
用户明确提出需要“术前术后对比图”，并特别指出了“玻尿酸”（填充/体积变化）和“祛痘祛斑”（肤质/纹理变化）两个场景。这实际上代表了计算机视觉领域中两个截然不同的技术路径：纹理合成 (Texture Synthesis) 与 几何形变 (Geometric Deformation)。
3.1.1 场景一：祛痘祛斑与肤质改善 (Texture Engine)
 * 技术挑战： 简单的模糊或磨皮算法会丢失皮肤的毛孔、绒毛等高频细节，导致生成的图片呈现“假面感”或“塑料感”，无法令求美者信服。
 * 解决方案：生成对抗网络 (GAN) 的应用
   我们将采用改进版的 SkinGAN 或带有编码器的 StyleGAN3 架构 。
   * 病灶检测 (Detection)： 首先利用基于 U-Net 的分割网络，精确识别面部的痤疮（Acne）、色斑（Pigmentation）、红血丝（Rosacea）等区域。该模型需在包含数万张标注皮肤病理图像的数据集上进行训练，以达到 >95% 的像素级分割精度。
   * 智能修复 (Inpainting)： 识别出的病灶区域作为掩码 (Mask) 输入到生成器中。生成器利用上下文注意力机制 (Contextual Attention)，从图像中未受损的健康皮肤区域提取纹理特征，合成出自然的皮肤组织填补病灶区域。
   * 细节保留 (Detail Preservation)： 引入感知损失函数 (Perceptual Loss) 和纹理一致性损失，强制模型保留原有的面部光影结构和毛孔分布，确保修复后的皮肤看起来是“治愈了”而不是“被PS了”。
   * 程度可控 (Adjustability)： 系统提供 0-100% 的“治疗程度”滑块，模拟从单次治疗到疗程结束的渐进效果，管理客户期望值。
3.1.2 场景二：玻尿酸填充与轮廓塑形 (Volumetric Engine)
 * 技术挑战： 玻尿酸注射本质上是面部软组织的体积增加。2D 图像的液化（Warping）技术无法正确模拟体积增加后的光影变化（如苹果肌饱满后高光点的位移），且容易导致背景扭曲。
 * 解决方案：3D 可形变模型 (3DMM) 与可微渲染
   我们将构建基于 FLAME (Faces Learned with Articulated Model and Expressions) 的 3D 重建管线 。
   * 单图重建 (3D Reconstruction)： 用户上传单张 2D 正面或侧面照片，系统利用深度神经网络回归出人脸的 3D 形状参数、表情参数、纹理参数及光照参数，重建出高保真的 3D 面部网格 (Mesh)。
   * 体积模拟 (Volumetric Simulation)： 建立玻尿酸注射的物理仿真模型。当用户选择“填充法令纹 1ml”时，系统不在像素层面操作，而是在 3D 网格的特定解剖层（如深层脂肪垫）增加相应的体积向量。系统需内置“物理约束层” ，依据皮肤弹性模量限制形变幅度，防止生成违反人体解剖学结构的夸张效果（如“蛇精脸”），确保模拟结果的医学合理性。
   * 光影重绘 (Re-lighting)： 变形后的 3D 模型经过可微渲染器（Differentiable Renderer）重新投射回 2D 平面。由于 3D 几何结构发生了改变，渲染器会根据原始光照参数重新计算漫反射和高光，使得填充部位呈现出真实立体的饱满感，而非简单的平面拉伸。
3.2 详细功能规格 (Functional Specifications)
FS-01: 多模态影像采集与质检
 * 输入支持： 支持来自单反相机（RAW/JPG）、专业皮肤检测仪（如 VISIA 导出数据）及移动端拍摄的高清图片。
 * 智能质检 (Smart QC)： 上传时系统自动检测图像质量。若存在光线过暗、面部遮挡、焦距模糊或非正脸角度等问题，AI 将实时反馈并指导用户重新拍摄。例如：“检测到左侧面部阴影过重，请调整光源”。
 * 自动对齐 (Auto-Alignment)： 无论原始拍摄角度如何，系统自动校正头部姿态，生成标准的“法兰克福平面”视角，便于前后对比。
FS-02: 交互式模拟工作台
 * 区域选择： 界面提供基于解剖分区的选择工具（如：额肌、眉间纹、鼻背、鼻唇沟、下颌缘等），而非简单的画笔涂抹。
 * 注射剂量模拟： 针对玻尿酸项目，提供以“支”或“毫升(ml)”为单位的调节滑杆。系统根据注射部位自动换算视觉变化量（例如：嘴唇注射 0.5ml 与 太阳穴注射 0.5ml 的视觉反馈截然不同）。
 * 分屏对比 (Split-Screen)： 提供“左右对比”、“重叠透视”、“滑动对比”三种视图模式。支持 4K 分辨率的实时渲染，延迟控制在 200ms 以内，保证咨询过程流畅不卡顿。
FS-03: 模拟结果导出与存证
 * 水印与免责声明： 所有导出的对比图自动添加机构 Logo 水印及“模拟效果仅供参考，实际效果因人而异”的法律免责声明文本。
 * 多格式交付： 支持生成静态对比图 (JPG)、动态变化 GIF 图及 3D 旋转视频 (MP4)，便于通过微信发送给客户。
3.3 用户体验流程 (User Journey)
为了清晰展示 FaceSim 模块的操作逻辑，我们将通过文字详细描述用户的操作旅程：
 * 启动与采集： 咨询师在 iPad 上打开 AesthetiCore 应用，选择“新建模拟”。调用后置摄像头，系统界面显示辅助构图线（眼睛水平线、面部中轴线）。咨询师拍摄客户正脸、左侧 45 度、右侧 45 度三张照片。
 * AI 预处理： 照片上传后，系统在 3 秒内完成 3D 重建与皮肤病灶分析。屏幕上标记出所有检测到的皮肤问题（红点标记痘印，蓝圈标记色斑）。
 * 方案定制：
   * 场景 A（皮肤）： 咨询师点击“一键净肤”，系统自动清除所有标记瑕疵。咨询师可手动调节“保留特征”，如保留标志性黑痣。
   * 场景 B（轮廓）： 咨询师切换至 3D 模式，选择“玻尿酸填充”。点击“太阳穴”区域，拖动滑块至“2ml”。屏幕上的 3D 模型太阳穴部位实时饱满，且高光位置随之移动。
 * 沟通与调整： 咨询师向客户展示 iPad，拖动对比滑块。客户表示“下巴太尖了”，咨询师微调下巴参数，减少填充量。
 * 定稿与发送： 双方确认模拟效果。点击“生成报告”，系统打包生成包含模拟图、预估注射量及报价的 PDF，一键发送至客户微信。
4. 模块二：品牌一致性设计引擎 (BrandGuard Module)
4.1 需求深度拆解
客户要求“每天做朋友圈海报”，且必须基于“美宜家”的 VI（Visual Identity）色系，风格需符合“医美行业风格”。这反映了医美私域运营的高频痛点：内容需求量大但设计产能不足，且外包或非专业设计容易导致品牌形象稀释。
通用 AI 绘图工具（如 Midjourney）虽然强大，但难以精准控制品牌色（Hex Code）和特定排版风格。BrandGuard 模块的核心任务是**“戴着镣铐跳舞”**，即在严格的品牌规范限制下进行无限的创意生成。
4.2 技术实现路径
4.2.1 定制化 LoRA 模型训练 (Model Fine-Tuning)
我们将采用 Low-Rank Adaptation (LoRA) 技术对 Stable Diffusion XL (SDXL) 或 Flux 基础模型进行微调 。
 * 数据集构建： 收集“美宜家”过去 1-2 年内的高质量海报、诊所实景图、模特图以及医美行业的高级感素材（如极简风格、柔和光影、医疗蓝/科技白配色）。数据集规模控制在 50-100 张精选图片。
 * 风格锁定： 训练专门的 <Meiyijia-Style> LoRA 模型。该模型将学习品牌的特定光影质感（如通透的皮肤质感）、构图习惯（留白比例）及色彩倾向。
 * 触发词工程： 系统后台将用户输入的自然语言描述（如“夏季补水”）自动转化为包含触发词的高级 Prompt（如 masterpiece, best quality, Meiyijia-Style, medical aesthetics, serene blue background, water splash, hydration concept），确保生成的底图风格统一。
4.2.2 矢量图形合成与 VI 色彩强制 (Vector Composition & Hex Enforcement)
单纯依靠 AI 生成文字和 Logo 往往效果不佳且无法编辑。我们将采用 “底图生成 + 矢量合成” 的混合管线 。
 * 底图生成层： 由 LoRA 模型生成符合品牌调性的背景图（Background）和主体视觉元素（如水滴、玻尿酸针管、抽象线条）。
 * 矢量合成层： 利用程序化设计引擎（如 Polotno API 或 Canvas），在生成的底图上叠加 SVG 格式的品牌 Logo、标准字体的营销文案及装饰性色块。
 * 色彩强制 (Strict Hex Enforcement)： 这是满足 VI 需求的关键。虽然 AI 生成的底图色调可能略有偏差，但所有叠加的矢量元素（标题、按钮、边框）将强制使用“美宜家”的标准色值（例如 #00A0E9 医美蓝, #FFFFFF 纯白, #F2F2F2 高级灰）。系统还会对 AI 生成的底图进行后处理滤镜校色，使其直方图向品牌色系靠拢。
4.3 详细功能规格
BG-01: 智能海报工厂
 * 每日灵感库： 系统根据当下的节气（如立秋）、节日（如情人节）、行业热点（如抗衰季）自动推荐 3-5 个海报主题。
 * 模板自适应： 提供多种布局模板（如单图大字版、多图拼贴版、科普长图版）。用户只需输入核心文案（如“限时特惠 ￥999”），系统自动排版，适配文字长度，保证不遮挡底图主体。
 * 元素替换： 支持“拖拽式”替换。用户可以将系统内置的标准产品图（如某品牌玻尿酸盒子）拖入海报，AI 自动进行光影融合 (Harmonization)，使其看起来像是原本就在场景中一样。
BG-02: 品牌卫士 (Brand Compliance Check)
 * VI 合规检测： 系统内置视觉识别算法，在海报导出前扫描画面。如果画面主色调偏离品牌色系超过设定阈值（如 >15%），或 Logo 变形、被遮挡，系统将弹出警告并阻止导出。
 * 违禁词审查： 集成医疗广告法违禁词库（如“第一”、“顶级”、“根治”、“无创”等敏感词）。OCR 模块扫描海报上的所有文字，高亮显示潜在违规词汇，降低合规风险。
BG-03: 私域适配与分发
 * 格式优化： 默认输出 1080x1920px（朋友圈全屏）及 1080x1080px（朋友圈九宫格）两种尺寸。
 * 压缩算法： 针对微信朋友圈的图片压缩机制进行反向优化，确保导出的小于 500KB 的图片在微信中显示清晰，无明显噪点或色偏。
5. 模块三：自动化汇报套件 (ReportFlow Module)
5.1 需求深度拆解
用户需要“每周做工作汇报 PPT”，且需“符合医美行业风格”。这涉及两个核心痛点：数据的收集与整理耗时，以及PPT 排版的美观度。传统的做法是人工从各个系统（SaaS、Excel、微信群）扒数据，再手动填入 PPT，不仅效率低且容易出错。ReportFlow 旨在实现从“数据源”到“演示文稿”的端到端自动化。
5.2 技术实现路径：RAG 与文档生成
5.2.1 数据聚合引擎 (Data Aggregation Engine)
 * 多源数据接入： 系统通过 API 接口对接诊所现有的 HIS 系统、CRM 系统（如 Aesthetic Record）、财务软件（如金蝶/用友）及考勤系统。
 * 指标计算： 自动化计算周度核心指标，如：
   * 初诊量/复诊量及其环比增长 (WoW)。
   * 成交率 (Conversion Rate)： 分渠道（大众点评、转介绍、自然到店）统计。
   * 单客价值 (ARPU)。
   * 项目占比： 手术类 vs 非手术类收入结构 。
 * 异常检测： 算法自动扫描数据，识别异常波动（如某日营收突降 50% 或 某医生投诉率激增），并将其标记为“需要解释的关键事件”。
5.2.2 智能 PPT 生成器 (SlideGen)
 * 结构化生成： 利用 LLM（大语言模型）基于本周数据生成汇报大纲。
   * 输入： 本周经营数据 JSON、上周遗留问题状态。
   * Prompt： “作为一家医美诊所的运营总监，请根据以下数据生成一份周报大纲，重点分析玻尿酸业绩下滑的原因，并提出下周改进计划。”
   * 输出： 包含 5-8 页的 PPT 结构（封面、核心业绩概览、项目分析、客户反馈、下周计划）。
 * 原生图表注入： 不同于简单的截图，系统利用 python-pptx 库，将数据直接写入 PowerPoint 的原生图表对象（Chart Object）。这意味着生成的 PPT 中的柱状图、饼图是可编辑的，用户可以在 PPT 中直接修改数据源或调整颜色，极大提升了其实用性。
 * 风格渲染： 调用“美宜家”专属的 PPT 母版（Master Slide）。确保所有标题字体、正文行距、背景色块、图表配色严格遵循 VI 规范。LLM 生成的总结性文字（如“本周营收稳中有升，得益于双十一预售活动的开启”）会自动填充到对应的文本框中。
5.3 详细功能规格
RF-01: 一键周报 (One-Click Weekly Report)
 * 触发机制： 用户点击“生成本周汇报”，系统自动拉取本周一至周日的数据。
 * 智能摘要： AI 自动生成“本周高光时刻”（如：王医生单日业绩破纪录）和“预警信息”（如：新客占比跌破 20%）。
 * 编辑与微调： 生成的 PPT 支持在线预览和编辑。用户可以手动修改 AI 生成的结论，或增删页面，确认无误后下载.pptx 文件。
RF-02: 专项分析报告 (Ad-hoc Analysis)
除了周报，系统支持生成专项报告。例如输入“分析过去三个月皮秒激光项目的表现”，系统自动生成包含通过率、复购率、设备空置率等维度的深度分析 PPT。
6. 模块四：智能治疗方案与动态定价 (ConsultAI Module)
6.1 需求深度拆解
用户需求是“基于客户当下情况和特色项目，设计面部变美方案及价格”。这要求系统具备医疗逻辑的推理能力和商业逻辑的计算能力。这实际上是一个 CPQ (Configure, Price, Quote) 系统在医美垂直领域的应用。
6.2 医疗逻辑：基于规则的专家系统 (Rule-Based Expert System)
医美方案设计必须遵循医学原则，不能随意搭配。我们将构建一个基于规则的推荐引擎 。
 * 知识图谱构建： 将面部适应症与治疗项目建立关联。例如：
   * 适应症： 静态皱纹（眉间纹）。
   * 解决方案： 肉毒素除皱（Botox/衡力）。
   * 禁忌症： 重症肌无力、妊娠期。
 * 推理逻辑：
   * 输入： FaceSim 识别出的面部特征（如：鼻基底凹陷、咬肌肥大）、客户填写的问卷（如：预算范围、恢复期接受度、过敏史）。
   * 规则链： IF 咬肌肥大 AND 无备孕计划 THEN 推荐=瘦脸针; IF 鼻基底凹陷 THEN 推荐=玻尿酸填充 OR 假体植入。
   * 输出： 生成“分阶治疗方案”。
     * 基础版（经济型）： 解决最痛点问题（如仅除皱）。
     * 进阶版（推荐型）： 联合治疗（如除皱+填充），效果更佳。
     * 尊享版（全脸打造）： 包含光电抗衰等提升类项目。
6.3 商业逻辑：动态定价引擎 (Dynamic Pricing Engine)
医美行业的定价极其复杂，涉及医生职级、耗材成本、促销活动及库存压力。ConsultAI 将引入动态定价算法 。
6.3.1 定价因子模型
系统通过以下公式计算最终报价：

 * C_{base} (基础成本)：耗材费 + 房间占用费 + 护士工时费。
 * M_{target} (目标毛利)：诊所设定的最低利润额。
 * K_{doc} (医生系数)：
   * 普通医师 = 1.0
   * 主治医师 = 1.2
   * 院长/专家 = 1.5 
 * K_{time} (时段系数)：
   * 闲时（周二上午）= 0.9 (促销引流)
   * 忙时（周六下午）= 1.1 (溢价调节拥堵) 
 * K_{client} (客户系数)：
   * 新客首次体验 = 0.8 (低价拓客)
   * 会员等级折扣 = 0.85 - 0.95
   * 复杂修复案例 = 1.3 (风险溢价)
6.3.2 智能报价单 (Smart Quotation)
系统生成的报价单不再是一张冷冰冰的小票，而是一份 H5 互动页面：
 * 方案可视化： 每一项费用旁边配有该项目的原理动画（调用 MediaGen 素材）和 FaceSim 的模拟效果图。
 * 价格透明化： 清晰列出材料费、医生费、麻醉费，建立客户信任。
 * 即时激励： 页面底部显示“今日下单赠送术后修复面膜”或“限时 24 小时锁定价格”，利用紧迫感促进成交。
7. 模块五：财务智能仪表盘 (FinOps Module)
7.1 需求深度拆解
用户需要“每周财务数据分析，基于报表给出运营成本结构建议”。这表明用户需要的不仅是数据的展示 (Descriptive Analytics)，更是数据的诊断 (Diagnostic Analytics) 和 建议 (Prescriptive Analytics)。中国医美诊所成本结构复杂，高昂的房租、人力和营销费用是主要的利润杀手。
7.2 核心财务指标体系 (KPI Framework)
FinOps 模块将重点监控以下关键指标 ：
 * 运营利润率 (Operating Margin)： 监控整体盈利健康度。预警线设为 <15%。
 * 人效与坪效：
   * RevPAR (每房间每小时营收)： 手术室目标 ~$3,000/hr，治疗室目标 ~$600-$1,000/hr。低 RevPAR 意味着空间浪费。
   * Staff Utilization (人员利用率)： 医生和咨询师的工时饱和度。
 * CAC/LTV 比率： 获客成本 (CAC) 与 客户生命周期价值 (LTV) 的比值。健康模型应 > 3:1。若 < 1:1，说明在赔本赚吆喝。
 * 耗材占比 (COGS Ratio)： 尤其是高值耗材（如乔雅登、热玛吉探头）。若占比异常升高，可能存在库存损耗或私单风险。
7.3 成本结构建议引擎 (Cost Advisory Engine)
这是本模块的核心智能所在。系统不只显示“成本高了”，而是分析“为什么高”并给出“怎么办”。
 * 场景 A：营销费比过高
   * 诊断： 检测到本周营销费用占营收比重达到 50%（行业健康值为 30-40%），且新客转化率仅为 5%。
   * AI 建议： “营销效率低下。建议暂停‘通用关键词’竞价广告，将预算向‘品牌词’和‘老带新’活动倾斜。分析显示，转介绍客户的转化率是广告客户的 4 倍。”
 * 场景 B：人力成本闲置
   * 诊断： 检测到周二至周四下午，咨询师和治疗师的利用率低于 30%。
   * AI 建议： “由于周中闲时由于人力空置导致固定成本摊销过高。建议推出‘周中会员日’活动，针对脱毛、水光等标准化项目提供 85 折优惠，以填充闲置运力。”
 * 场景 C：库存积压风险
   * 诊断： 某品牌玻尿酸库存周转天数超过 90 天。
   * AI 建议： “库存预警。建议打包设计‘全脸填充套餐’或作为高客单价手术的赠品进行去库存处理，释放现金流。”
7.4 仪表盘可视化设计
8. 模块六：生成式视频工坊 (MediaGen Module)
8.1 需求深度拆解
用户需要“基于医生和机构给出专业视频生成提示词，用于生成有趣生动的项目演示模拟视频”。短视频（抖音/视频号）已成为中国医美市场最大的流量入口 。医生需要通过视频建立专业 IP，但往往缺乏脚本创作能力和拍摄时间。MediaGen 旨在降低视频生产门槛。
8.2 提示词工程库 (Prompt Engineering Library)
系统不仅是生成视频，而是提供一套结构化的 Prompt Builder，确保生成的视频既符合医学原理，又具备社交媒体的传播属性（黄金前3秒原则）。
8.2.1 视频模板分类
 * 科学原理演示类 (3D Animation Style)：
   * 应用场景： 解释热玛吉、超声炮等光电项目原理，或玻尿酸填充层次。
   * 提示词模板： [主体: 人体皮肤剖面图][动作: 胶原蛋白纤维受热收缩并增生][光效: 科技蓝与亮白色, 洁净明亮][镜头: 微距推进, 展现微观变化] 。
   * 价值： 用可视化的方式解释“为什么有效”，建立专业度。
 * 医生真人数字人 (Realistic Avatar Style)：
   * 应用场景： 科普口播、术后注意事项讲解。
   * 技术实现： 采集医生 5 分钟真视频，训练数字分身（Digital Avatar）。
   * 提示词模板： [主体: 亚洲女性皮肤科医生, 35岁左右, 穿着白大褂][动作: 面对镜头微笑讲解, 手势自然][背景: 高端诊所咨询室, 景深虚化][风格: 4K画质, 真实感, 影棚级布光] 。
 * 效果蜕变类 (Before/After Transition Style)：
   * 应用场景： 展示（模拟的）术后效果，激发购买欲。
   * 提示词模板： [主体: 女性面部特写][动作: 从暗沉粗糙肤质平滑过渡到水光玻璃肌][风格: 柔美转场, 美妆广告质感, 高调摄影][特效: 闪光粒子点缀] 。
8.3 功能规格
MG-01: 智能脚本撰写 (AI Scriptwriter)
 * 输入： 医生选择主题（如“肉毒素除皱”）、目标受众（如“25-35岁上班族”）、视频时长（如“60秒”）。
 * 生成： LLM 生成分镜脚本。
   * 0-3秒（黄金开头）： “你也有一笑就崩的鱼尾纹吗？别急，这三点你必须知道！”（痛点切入）。
   * 4-40秒（干货输出）： 解释皱纹成因及肉毒素原理（搭配 3D 演示画面）。
   * 41-60秒（引导转化）： “关注我，带你变美不踩坑。”（Call to Action）。
MG-02: 视频生成与合成
 * API 集成： 系统后端对接主流视频生成模型（如 Sora, Runway Gen-3）或国内的视频大模型（如快手“可灵”、阿里“EMO”）。
 * 自动剪辑： 将生成的视频片段、数字人口播片段、字幕、背景音乐自动合成。支持一键匹配热门 BGM，卡点剪辑，提升完播率。
9. 实施路径、合规与安全 (Implementation & Compliance)
9.1 数据安全与隐私合规
医美数据包含大量的人脸生物识别信息和医疗健康记录，属于极度敏感数据。AesthetiCore 将严格遵循中国《个人信息保护法 (PIPL)》及《数据安全法》。
 * 本地化存储： 所有中国大陆境内产生的数据，严格存储在境内服务器，严禁跨境传输。
 * 最小化采集原则： 仅采集实现功能所必须的数据。例如，FaceSim 仅上传面部数据，不强制关联身份证号。
 * 知情同意： 在采集客户照片前，系统会在 iPad 上弹出电子签署页面，明确告知数据用途（仅用于方案设计），并承诺不用于其他商业用途。
9.2 部署与实施路线图 (Roadmap)
考虑到系统复杂性，建议分三阶段实施：
 * 阶段一：视觉与效率（第 1-3 个月）
   * 上线 FaceSim (2D 版)：优先解决祛痘祛斑模拟，技术成熟度高，立竿见影提升咨询体验。
   * 上线 BrandGuard：规范全院海报输出，建立品牌视觉护城河。
   * 目标： 提升咨询转化率 10%，降低设计外包成本 80%。
 * 阶段二：数据与决策（第 4-6 个月）
   * 上线 ReportFlow 与 FinOps：打通数据孤岛，让管理者看到真实的经营状况。
   * 上线 ConsultAI (基础版)：实现基于规则的方案推荐，统一全院报价逻辑。
   * 目标： 提升管理效率，库存周转率提升 20%。
 * 阶段三：全域智能（第 7-12 个月）
   * 上线 FaceSim (3D 版)：攻克玻尿酸体积模拟难题，建立技术壁垒。
   * 上线 MediaGen：全面赋能医生 IP 孵化，通过短视频获取公域流量。
   * 目标： 打造区域标杆数字化诊所，构建私域+公域流量闭环。
10. 结语
AesthetiCore 系统的设计初衷，是回应中国医美行业对“精细化运营”的呼唤。它不只是一个生成图片的工具，而是一套涵盖了从视觉营销到医疗交付再到经营管理的完整操作系统。通过将美宜家的 VI 规范植入算法，将专家的诊断逻辑植入代码，将财务总监的分析思路植入仪表盘，AesthetiCore 致力于帮助医美机构在日益激烈的竞争中，通过技术手段实现降本、增效与合规，最终回归医疗服务的长期主义价值。
